# Localization Excel Updater 更新说明与用法

## Skill 导出内容

这个导出包包含：

- `localization-excel-updater/`
- `更新说明与用法（中文）.md`

将 `localization-excel-updater/` 整个文件夹复制到：

```text
~/.codex/skills/localization-excel-updater
```

然后重启 Codex 即可加载。

## 这次更新了什么

### 1. 支持修改已有 key 的文案

现在不只是能新增字段，还能修改指定端、指定文件里已经存在的 `key`。

特点：

- `key` 保持不变
- 可以通过 `key` 定位
- 也可以通过“原中文文案”定位
- 修改中文后，会同步更新该行已有语言列
- 会继续同步飞书表格
- 会继续执行 `run.command`
- 会继续提交并推送 Git

### 2. 支持下载飞书表格到本地

现在可以把：

- `app端`
- `眼镜端`
- `全部`

对应的飞书表格下载成 `.xlsx` 到本地目录。

这个流程只下载，不改 Git。

### 3. 支持从飞书回拉并上传 Git

现在可以把飞书中的表格覆盖回本地 `excel_files/*.xlsx`，然后执行生成和 Git 推送。

适合“飞书是临时最新版本”的场景。

### 4. 支持新建 workbook

现在可以：

- 在本地创建新的 `.xlsx`
- 在飞书创建同名表格
- 写入标准表头
- 然后执行推送

### 5. Git 推送后显示 GitLab 可点击链接

每次成功 push 后，会在回复中显示：

```text
仓库: http://...
本次提交: http://.../-/commit/xxxx
```

### 6. 新增或修改成功后推送飞书群消息

在 `add_rows` 或 `edit_rows` 成功完成后，会向飞书群发送结果通知，内容包括：

- 端
- 文件
- 最近新增或修改的 `key`
- 对应中文文案
- 最近提交链接
- 飞书文档链接

### 7. 转义字符按原样写入表格

这是最新补充的一条规则。

如果你在文案里输入：

- `\n`
- `\t`
- `\\`
- `\"`

这类内容，skill 会把它们按字面量原样写进单元格，不会自动转成真实换行、Tab 或其他转义结果。

除非你明确要求“把 `\n` 转成换行”，否则默认都按原样保留。

### 9. 英文和繁体中文规则进一步收紧

这是这次重点修复的规则：

- `en` 默认只允许整句首字母大写
- 除品牌名、产品名、缩写、或你明确指定的大小写要求外，其余单词都应保持小写
- `zh-rHK` 必须按香港繁体风格翻译
- `zh-rTW` 必须按台湾繁体风格翻译
- `zh-rHK` 和 `zh-rTW` 都不允许混入简体中文

也就是说，之后如果出现：

- 英文里很多单词都大写
- `zh-rHK` / `zh-rTW` 中出现简体字
- 香港繁体、台湾繁体风格混用

都应该视为不符合 skill 规则，需要改正后再写入表格。

### 8. 支持把 skill 自身更新同步到指定 Git 仓库

现在这套 skill 不只是能处理多语言表格，也支持把它自己的最新版本导出并提交到你指定的 Git 仓库。

适用场景：

- 想备份这套 skill
- 想把 skill 发给同事
- 想持续在 GitHub / GitLab 上维护版本

处理效果：

- 从当前安装目录导出最新 `localization-excel-updater`
- 可同时附带中文更新说明
- 同步到你提供的本地 Git 仓库
- 自动 `git add / commit / push`
- 成功 push 后，会额外给出一段“本次提交描述（中文版）”

## 最重要的新功能：修改已有 key 的文案

### 用法 1：按 key 修改

这是最推荐的写法。

```text
用 Localization Excel Updater

端：app端
文件：home
key：not_connected
改成：未连接设备
```

适用场景：

- 你知道准确的 `key`
- 想直接改这一行

处理效果：

- 保留 `key=not_connected`
- 把 `zh` 改成新文案
- 自动更新这行已有语言列
- 同步飞书
- 生成资源
- 提交并推送 Git

### 用法 2：按原中文修改

当你不知道 `key` 时可以这样写：

```text
用 Localization Excel Updater

端：app端
文件：home
原文：未连接
改成：未连接设备
key 不变
```

适用场景：

- 你只知道现在表里的中文
- 不知道对应 `key`

处理逻辑：

- 优先找 `zh` 完全匹配的行
- 如果找不到，再尝试唯一的包含匹配
- 找到后保留原 `key`

### 用法 3：强调翻译约束

如果你希望修改后保持术语统一，可以直接补一句：

```text
翻译要求：请保持长句里的“未连接”和这个字段翻译一致
```

或者：

```text
翻译要求：所有语言统一使用小写
```

### 用法 4：包含字面量转义字符

如果你要把 `\n` 当普通文本写进表格，请直接这样发：

```text
用 Localization Excel Updater

端：app端
文件：prompter
key：settings_auto_scroll_desc
改成：提词内容读到哪儿滚到哪儿，有网和无网都可用\n当前仅支持中文和英文\n使用时请保持眼镜和手机的距离在6米以内
```

现在 skill 会把 `\n` 原样保存到单元格，而不是替你变成真实换行。

## 其他常用用法

### 发布 skill 到 Git 仓库

```text
把这个 skill 更新到我给的 Git 仓库
本地仓库：/path/to/repo
```

或者：

```text
把 Localization Excel Updater 导出并推到这个仓库：
git@github.com:xxx/yyy.git
```

以后这类发布完成后，除了仓库链接和提交号，还会附带：

```text
本次提交描述（中文版）：
1. 更新了什么
2. 为什么改
3. 这次新增了哪些能力或规则
```

说明内容会根据实际更新点来写，不会只给一个笼统结论。

### 新增字段

```text
用 Localization Excel Updater

端：app端
文件：home
字段：
眼镜 Wi-Fi 打开但未连接，建议关闭
未连接
```

### 下载飞书表格

```text
用 Localization Excel Updater
下载飞书 app端
```

或：

```text
用 Localization Excel Updater
下载飞书 全部
```

### 从飞书回拉并上传 Git

```text
用 Localization Excel Updater
下载app端home并上传 git
```

或更明确一点：

```text
用 Localization Excel Updater
从飞书回拉 app端 并上传 git
```

### 创建新表格

```text
用 Localization Excel Updater

端：app端
文件：sdk_new
创建表格
```

## 备注

- 如果飞书同步失败，skill 会在 `run.command` 前停止。
- 如果 Git push 失败，skill 会告诉你“本地已提交，但远端未推送成功”。
- 修改功能默认保留 `key` 不变，不支持直接重命名 `key`。
